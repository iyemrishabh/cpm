# Contributing to CPM

First off, thank you for considering contributing to CPM! It's people like you that make the open source community such a great place to learn, inspire, and create.

## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md). Please read it before interacting with the community.

## How Can I Contribute?

### Reporting Bugs

If you find a bug, please check the existing issues first. If it hasn't been reported yet, open a new issue. Include:
- A clear, descriptive title.
- Steps to reproduce the issue.
- Expected vs actual behavior.
- Environment details (OS, version, etc.).

### Suggesting Enhancements

We welcome new ideas! To suggest an enhancement, please open an issue describing the proposed feature, the problem it solves, and potential implementation details.

### Pull Requests

1. Fork the repository and create your branch from `main`.
2. Keep your branch updated with the upstream `main`.
3. If you've added code that should be tested, add tests.
4. Ensure the test suite passes.
5. Reference any relevant issues in your pull request description.
6. Commit messages should follow clear, concise conventions (e.g., [Conventional Commits](https://www.conventionalcommits.org/)).

## Code Style & Standards

Please ensure your code conforms to standard style guides for the language/framework you are contributing to (e.g. Prettier for JS/TS, PEP8 for Python, gofmt for Go). Run linting and formatting commands prior to submitting PRs.

Thank you!
