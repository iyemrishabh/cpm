# 1. Overview

This document identifies the stakeholders of Cloud Platform Manager (CPM), their responsibilities, expectations, and interactions with the platform.

Understanding stakeholders ensures that the platform is designed to meet the needs of everyone involved in developing, operating, securing, and consuming cloud infrastructure.

---

# 2. Primary Stakeholders

## Platform Engineers

### Responsibilities

- Design the cloud platform
- Build reusable platform components
- Define platform standards
- Manage platform lifecycle
- Improve developer experience

### Expectations

- Automated deployments
- Component version management
- Dependency-aware orchestration
- Reliable upgrades and rollbacks
- Platform observability

---

## Cloud Engineers

### Responsibilities

- Deploy AWS infrastructure
- Manage AWS services
- Troubleshoot cloud issues
- Support application teams

### Expectations

- Simplified infrastructure deployment
- Standardized AWS accounts
- Faster provisioning
- Reduced manual work

---

## DevOps Engineers

### Responsibilities

- CI/CD pipelines
- Infrastructure automation
- Deployment management
- Release coordination

### Expectations

- API-driven automation
- GitHub Actions integration
- Reliable deployment workflows
- Easy rollback mechanisms

---

## Security Engineers

### Responsibilities

- Security governance
- IAM reviews
- Compliance monitoring
- Audit management

### Expectations

- Secure platform defaults
- Centralized audit logs
- Policy enforcement
- Compliance reporting

---

## FinOps Engineers

### Responsibilities

- Cost monitoring
- Cost optimization
- Resource utilization analysis

### Expectations

- Cost visibility
- Resource inventory
- Optimization recommendations
- Platform cost reporting

---

## Cloud Architects

### Responsibilities

- Platform architecture
- Technology selection
- Governance strategy
- Infrastructure standards

### Expectations

- Modular architecture
- Scalable design
- Extensible platform
- Standardized components

---

# 3. Secondary Stakeholders

## Application Teams

Application teams consume the platform to deploy workloads.

### Expectations

- Reliable platform
- Faster onboarding
- Stable environments
- Self-service capabilities (future)

---

## Project Managers

Project managers monitor platform delivery and progress.

### Expectations

- Predictable releases
- Project visibility
- Delivery metrics

---

## Organization Leadership

Leadership expects the platform to improve operational efficiency while reducing risk and cost.

### Expectations

- Standardization
- Security
- Scalability
- Cost optimization
- Faster delivery

# 6. Summary

Cloud Platform Manager serves multiple engineering and business stakeholders.

By providing a centralized, automated, and standardized platform, CPM enables teams to collaborate more effectively while improving operational efficiency, security, governance, and cloud platform reliability.
