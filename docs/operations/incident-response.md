# Incident Response Policy

This document defines the process for identifying, responding to, and resolving security or operational incidents.

## Incident Severity Levels

- **SEV-1 (Critical)**: Core service is down, critical data loss, or active security breach.
- **SEV-2 (High)**: Major service disruption with workaround available.
- **SEV-3 (Medium)**: Non-critical bugs, minor issues.

## Response Checklist

1. **Detect**: Alert triggered or customer report received.
2. **Triage**: Determine severity level and assign Incident Commander.
3. **Mitigate**: Take actions to restore service or isolate security issue.
4. **Communicate**: Update status page and stakeholders every 30 minutes for SEV-1.
5. **Resolve**: Confirm system is stable and clean.
6. **Post-Mortem**: Document root cause and list preventive actions within 5 business days.
